# Frontend Multi-Language (i18n) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add English + Bengali UI translation to the `fe/` app with a header language switcher, browser auto-detect, and persisted choice, using `react-i18next`.

**Architecture:** `react-i18next` is initialized once in `fe/src/lib/i18n.ts` with static JSON resources and a localStorage+navigator language detector. Components call `t("key")`; a `LanguageSwitcher` toggles `i18n.changeLanguage`. Strings live in `fe/src/locales/{en,bn}/translation.json`; English is the source of truth and missing Bangla keys fall back to English.

**Tech Stack:** React 18 + TypeScript + Vite, `i18next`, `react-i18next`, `i18next-browser-languagedetector`, shadcn/ui.

## Global Constraints

- Launch languages: `en` (default) and `bn`; architected so adding a language = new JSON file + one line in `supportedLngs`
- Scope is **UI text only** — never wrap user-generated content (review/comment text, product names, usernames) or DB-provided category labels / static-page bodies in `t()`
- English (`en/translation.json`) is the source of truth: every key MUST exist in `en`; Bangla is best-effort and falls back to English
- `fallbackLng: "en"`, `supportedLngs: ["en", "bn"]`, detection order `["localStorage","navigator"]`, cache `["localStorage"]`, localStorage key `lang`
- `<html lang>` must reflect the active language
- Keys are nested and grouped by area (`nav`, `common`, `auth`, `footer`, `home`, `review`, …); plurals use i18next `_one`/`_other` suffixes
- No test suite exists — verify each task with `cd fe && npm run build` (must pass) and manual toggle
- All API calls keep using `apiFetch`; this feature touches presentation only
- CMS app (`cms/`) is out of scope

- [ ] **Step 4: Create `fe/src/lib/i18n.ts`**

```ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";
import en from "@/locales/en/translation.json";
import bn from "@/locales/bn/translation.json";

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: en },
      bn: { translation: bn },
    },
    fallbackLng: "en",
    supportedLngs: ["en", "bn"],
    interpolation: { escapeValue: false }, // React already escapes
    detection: {
      order: ["localStorage", "navigator"],
      caches: ["localStorage"],
      lookupLocalStorage: "lang",
    },
  });

// Keep <html lang> in sync for accessibility + SEO.
const applyLang = (lng: string) => {
  document.documentElement.lang = lng?.startsWith("bn") ? "bn" : "en";
};
i18n.on("languageChanged", applyLang);
applyLang(i18n.language || "en");

export default i18n;
```

- [ ] **Step 2: Create `fe/src/components/LanguageSwitcher.tsx`**

```tsx
import { useTranslation } from "react-i18next";
import { Globe, Check } from "lucide-react";
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuTrigger,
} from "@/components/ui/dropdown-menu";
import { Button } from "@/components/ui/button";

const LANGS = [
  { code: "en", labelKey: "language.english" },
  { code: "bn", labelKey: "language.bangla" },
] as const;

export function LanguageSwitcher() {
  const { t, i18n } = useTranslation();
  const current = i18n.language?.startsWith("bn") ? "bn" : "en";

  return (
    <DropdownMenu>
      <DropdownMenuTrigger asChild>
        <Button
          variant="ghost"
          size="icon"
          className="rounded-full h-9 w-9"
          aria-label={t("language.change")}
        >
          <Globe className="h-4 w-4" />
        </Button>
      </DropdownMenuTrigger>
      <DropdownMenuContent align="end" className="rounded-2xl">
        {LANGS.map((l) => (
          <DropdownMenuItem
            key={l.code}
            className="cursor-pointer rounded-xl"
            onClick={() => i18n.changeLanguage(l.code)}
          >
            <span className="flex-1">{t(l.labelKey)}</span>
            {current === l.code && <Check className="h-4 w-4 ml-2" />}
          </DropdownMenuItem>
        ))}
      </DropdownMenuContent>
    </DropdownMenu>
  );
}
```

### Task 4: Translate high-traffic pages

**Files:**
- Modify: `fe/src/pages/Index.tsx`, `fe/src/pages/BrowseReviews.tsx`, `fe/src/pages/Auth.tsx`, `fe/src/pages/ReviewDetails.tsx`, `fe/src/pages/Profile.tsx`, `fe/src/pages/OwnerDashboard.tsx`

- May touch shared components these pages render (e.g. `ReviewCard.tsx`, `CategoryFilter.tsx`) — translate any literal UI text found there too.

**Interfaces:**
- Consumes: i18n instance; adds key groups `home`, `browse`, `auth`, `review`, `profile`, `owner`

**The migration recipe (apply to each file):**
1. Read the file.
2. Add `import { useTranslation } from "react-i18next";` and `const { t } = useTranslation();` (and `i18n` if a date/locale is needed).
3. For every **user-facing UI literal** — JSX text, `placeholder`, `aria-label`, `title`, button labels, headings, empty-state copy, toast `title`/`description`, zod/validation messages — replace with `t("group.key")` and add the key to BOTH `en` and `bn` JSON files under the page's group.
4. For dynamic values use interpolation: `t("review.likes", { count })` with plural keys `likes_one`/`likes_other`.
5. For dates: replace `new Date(x).toLocaleDateString("en-US", {...})` with `new Date(x).toLocaleDateString(i18n.language === "bn" ? "bn-BD" : "en-US", {...})`.
6. Do NOT translate: review/comment text, product names, usernames, category labels from the API.
7. Provide a best-effort Bangla value for every new key (English is the fallback if unsure).

**Worked example (BrowseReviews.tsx):**

```tsx
// before
<h1 className="...">Browse Reviews</h1>
<p className="...">Discover honest reviews from our community</p>
// ...
<SelectItem value="latest">Latest</SelectItem>
// ...
<h3 className="...">No reviews found</h3>

// after
<h1 className="...">{t("browse.title")}</h1>
<p className="...">{t("browse.subtitle")}</p>
// ...
<SelectItem value="latest">{t("browse.sortLatest")}</SelectItem>
// ...
<h3 className="...">{t("browse.noResults")}</h3>
```

with, in `en/translation.json`:

```json
  "browse": {
    "title": "Browse Reviews",
    "subtitle": "Discover honest reviews from our community",
    "sortLatest": "Latest",
    "sortPopular": "Most Popular",

    "sortComments": "Most Discussed",
    "ratingAll": "All Ratings",
    "noResults": "No reviews found",
    "noResultsHint": "Try adjusting your search or filters"
  }
```

and the matching Bangla group in `bn/translation.json`.

## Notes

- **Bangla quality:** translations are best-effort and centralized in
  `bn/translation.json` for easy human refinement later. English is always the
  fallback, so an imperfect or missing Bangla string never breaks the UI.
- **Adding a 3rd language later:** add `fe/src/locales/<code>/translation.json`,
  import it in `i18n.ts`, add `<code>` to `supportedLngs`, and add it to the
  `LANGS` array in `LanguageSwitcher.tsx`.
- **No backend/CMS changes** are part of this plan.
